# API reference

The canonical API reference is the **interactive OpenAPI documentation** auto-generated by FastAPI from the source code. It is always in sync with the actual implementation.

| Format | URL | Description |
|--------|-----|-------------|
| **Swagger UI** | [`/docs`](http://localhost:8000/docs) | Interactive — try endpoints directly from the browser |
| **ReDoc** | [`/redoc`](http://localhost:8000/redoc) | Read-only — cleaner layout for browsing |
| **OpenAPI JSON** | [`/openapi.json`](http://localhost:8000/openapi.json) | Machine-readable spec for code generators, Postman, etc. |

> These are served by the running app. Replace `localhost:8000` with your deployment URL.

## Endpoint overview

A quick-reference table. See the Swagger UI for full request/response schemas.

### Pages

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/` | Server-rendered roadmap page (`?department=`, `?product=`, `?cycle=`) |
| `GET` | `/login` | Redirect to OIDC provider (automatic) |
| `GET` | `/callback` | OIDC callback (exchanges auth code for session) |
| `GET` | `/token` | Display session cookie for `curl` usage |

### Sync

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/v1/sync` | Trigger a background Jira sync |
| `GET` | `/api/v1/status` | Current sync status + DB row counts |
| `GET` | `/api/v1/sync/schedule` | Scheduler timing info |

### Roadmap items

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/v1/roadmap` | JSON list of items (`?product=`, `?status=`, `?release=`) |

### Products

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/v1/products` | List all products with Jira source mappings |
| `GET` | `/api/v1/products/{id}` | Get a single product |
| `POST` | `/api/v1/products` | Create a product |
| `PUT` | `/api/v1/products/{id}` | Replace a product (full replacement) |
| `DELETE` | `/api/v1/products/{id}` | Delete a product (unlinks items) |

### Snapshots

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/v1/snapshots` | List snapshot dates with item counts |
| `GET` | `/api/v1/snapshots/diff` | Compare two snapshots (`?from_date=`, `?to_date=`) |

### Cycles

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/v1/cycles` | List all cycles with state |
| `POST` | `/api/v1/cycles/{cycle}` | Register a new cycle |
| `PUT` | `/api/v1/cycles/{cycle}` | Change a cycle's state |
| `DELETE` | `/api/v1/cycles/{cycle}` | Remove a cycle |
| `GET` | `/api/v1/cycles/{cycle}/items` | Get frozen cycle items |

## Authentication

When OIDC is enabled (`OIDC_CLIENT_ID` is set):

- **Browser requests** → redirect to `/login`
- **API requests** (`/api/*`) → `401 {"detail": "Authentication required"}`

When OIDC is disabled (default for local dev), all routes are open.

To call API endpoints with `curl` when auth is enabled, visit `/token` in your browser after logging in to get a ready-to-copy cookie command.

See [Configuring authentication](../how-to/configuring-authentication.md) for setup details.
